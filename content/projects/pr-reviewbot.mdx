## 배경

혼자 진행하는 프로젝트에서는 PR을 올려도 리뷰해줄 사람이 없었습니다. AI를 활용해 PR 리뷰어 역할을 자동화할 수 있을지 실험해보고 싶었고, 동시에 GitHub App과 Webhook 동작 방식을 학습하고자 했습니다.

## 접근 방식

- GitHub App + Webhook으로 PR 이벤트 자동 감지
- PR diff 기반으로 변경 사항만 추출해 분석
- LLM(OpenAI API)으로 리뷰 생성 (모범 사례 / 개선 제안 형식으로 구조화)
- Commit SHA 기반 중복 리뷰 방지

## 회고

diff 단위가 아닌 코드베이스 전체를 이해한 상태에서 리뷰하는 방식을 시도해보고 싶습니다. 변경된 코드만 보는 것과 전체 맥락을 아는 것은 리뷰 품질에서 차이가 클 것 같습니다.
