## 배경

통합 CMS는 Next.js 앱 산출물이 서버 배포 단위에 함께 포함되는 구조였습니다. 프론트엔드 수정만으로도 전체 앱 빌드와 서버 재빌드가 필요했고, 앱이 늘어날수록 빌드 시간이 지속적으로 증가했습니다.

분석 결과 Next.js 앱의 순차 빌드와 공통 의존성 전파 구조가 주요 병목으로 확인됐습니다.

## 접근 방식

병목 영역별로 단계적인 개선을 진행했습니다.

먼저 Turborepo 기반 병렬 빌드를 도입하고 실패 시 자동 재시도 로직을 구성했습니다. turbo.json의 캐시 전략을 최적화하고 Docker 빌드 환경도 개선했습니다.

번들 사이즈 개선을 위해 Lottie 경량 빌드를 적용해 1.6MB에서 400KB로 약 75% 줄였고, lodash는 개별 import 및 네이티브 API 대체를 통해 추가로 감축했습니다.
