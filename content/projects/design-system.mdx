## 배경

브랜디, 하이버 등 사내 서비스에서 공통으로 사용할 UI 컴포넌트를 구축하는 업무를 맡았습니다. 서비스마다 유사한 컴포넌트가 중복 구현되는 비효율과 서비스 간 UI 불일치 문제가 있었습니다.

## 협업

디자이너와 컴포넌트 인터페이스를 함께 정리하는 과정이 가장 중요했습니다. Storybook을 디자이너-개발자 공용 언어로 활용해 컴포넌트의 모든 변형(variant, size, state)을 시각화하고 검토할 수 있도록 했습니다.

## 접근 방식

- Storybook 기반 디자인 시스템 구축
- TypeScript 타입 정의 및 컴포넌트 인터페이스 설계
- Rollup 기반 라이브러리 빌드 환경 구성
- AWS CodeArtifact 기반 사내 npm 패키지 배포

## 인사이트

디자인 시스템의 핵심은 컴포넌트 자체가 아니라 prop 인터페이스 규약이라는 점을 체감했습니다. Storybook은 단순 문서가 아니라 디자이너-개발자 사이의 공용 언어로 작동할 때 가장 효과적이었습니다.
