## 배경

광고 상품 결제를 위한 PG 연동 과정에서 결제 상태 처리 복잡도가 컸습니다. 광고 API의 응답값은 단독으로 결제 상태를 판단할 수 없어 여러 값의 조합이 필요했고, 화면마다 필요한 상태가 달랐습니다.

주문서, 결제 완료, 주문 상세 화면에서 각각 비슷하지만 다른 분기 로직이 중복 구현되는 문제가 있었습니다.

## 설계 원칙

복잡한 도메인 로직이 화면 컴포넌트에 그대로 들어가면 화면이 늘어날수록 유지보수 부담이 커집니다. "도메인 로직과 화면을 분리"하는 원칙으로 공통 매핑 레이어를 두기로 했습니다.

## 접근 방식

화면별로 흩어진 결제 상태 분기 로직을 공통 매핑 모듈로 분리했습니다.

광고 API의 상태 응답값과 추가 값을 조합해 화면별로 필요한 상태로 매핑하는 레이어를 두고, 화면 컴포넌트에서는 매핑된 상태값만 사용하도록 단순화했습니다. 결제 요청 후에는 redirectUrl 기반 폴링 방식으로 결제 상태를 동기화했습니다.

## 인사이트

복잡한 도메인 로직을 화면에서 분리해 매핑 레이어로 추상화하면 확장성과 안정성을 동시에 확보할 수 있습니다. 결제 흐름이 바뀌어도 매핑 정의만 수정하면 모든 화면에 일관되게 반영되는 구조가 운영 효율을 높였습니다.
