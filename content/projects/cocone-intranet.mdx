## 배경

코코네 사내에서 사용할 인트라넷 시스템을 새로 구축하는 업무를 맡았습니다. 단순히 화면만 만드는 것이 아니라, 향후 사내 다른 프로젝트에서도 재사용할 수 있는 공통 개발 환경 정립이 함께 필요했습니다.

## 설계 원칙

신규 프로젝트 시작 때마다 반복되던 환경 세팅, 코드 컨벤션, CI/CD 구성을 한 번에 정립해 다른 사내 프로젝트에서도 동일한 출발점에서 시작할 수 있도록 만드는 것을 목표로 했습니다.

또한 백엔드 스펙 변경에 안전하게 대응하기 위해 API 타입을 자동 생성하는 구조를 함께 도입했습니다.

## 접근 방식

- **사내 공통 보일러플레이트 구축**: Next.js + TypeScript + ESLint + Prettier + Husky 기반으로 코드 품질 자동 검사 환경을 갖춘 보일러플레이트 정립
- **API 타입 자동 생성**: Swagger 기반 openapi-typescript-codegen을 도입해 백엔드 스펙 변경 시 타입이 자동으로 갱신되는 구조 구성
- **CI/CD 파이프라인**: Jenkins + GitLab Webhook 기반 자동 배포 파이프라인 구축

## 협업 - 코드 리뷰 문화 도입

당시 사내에 코드 리뷰 문화가 정립되어 있지 않은 상태였습니다. 코드 품질과 지식 공유 측면에서 코드 리뷰가 필요하다고 판단해 GitLab Merge Request 기반의 코드 리뷰 문화 도입을 제안했습니다.

- MR(Merge Request) 기반 리뷰 프로세스 정착
- 최소 1명 이상의 Approve가 있어야 머지 가능하도록 GitLab 머지 정책 설정
- 리뷰 코멘트로 코드 컨벤션, 아키텍처 의견 공유

## 인사이트

보일러플레이트와 코드 리뷰 같은 인프라성 작업은 한 프로젝트만의 효과로 끝나지 않고, **이후 들어올 모든 프로젝트의 출발점을 결정**한다는 점을 체감했습니다. 짧은 기간의 작업이었지만 이런 기반 정립이 장기적으로 가장 큰 ROI를 만든다는 점을 학습했습니다.
