## 배경

외부 제휴 업체와의 계약 종료로 펜션 홈페이지를 내부에서 직접 제작하고 운영해야 했습니다. 운영자가 개발자에게 매번 요청하지 않고도 홈페이지를 만들고 수정할 수 있는 도구가 필요했습니다.

또한 여러 펜션 홈페이지를 운영해야 했기 때문에, 사이트가 늘어날 때마다 서버 운영 부담이 커지지 않는 구조가 중요했습니다.

## 설계 원칙

편집 화면과 실제 배포 결과물을 분리했습니다.

운영자는 웹빌더에서 드래그앤드롭 방식으로 페이지를 편집하고, 게시할 때는 완성된 페이지를 HTML/CSS/JSON 파일로 만들어 S3에 저장하도록 설계했습니다.

즉, 편집은 자유롭게 할 수 있게 만들고, 실제 사용자에게 보여지는 페이지는 별도 서버 운영 부담이 적은 정적 파일 형태로 제공하는 구조입니다.

## 접근 방식

- GrapesJS 기반으로 펜션 홈페이지 제작에 맞는 편집 화면 구축
- 템플릿, 섹션, 위젯을 재사용 가능한 단위로 구성
- About, Rooms, Special, Guide, Notice 등 기본 페이지를 템플릿 선택 시 자동 생성
- 도메인별로 편집 데이터를 저장하고 다시 불러오는 구조 구현
- 배포 시 각 페이지를 HTML 파일로 만들고, 이미지 등 정적 리소스와 함께 S3에 업로드

## 인사이트

GrapesJS 의 기본 기능을 그대로 쓰는 것보다, 실제 운영 방식에 맞게 줄이고 바꾸는 일이 중요했습니다.

특히 홈페이지를 매번 서버에서 렌더링하지 않고 정적 파일로 저장하니, 여러 사이트를 운영할 때 비용과 관리 부담을 줄일 수 있었습니다. 비개발자가 직접 수정할 수 있는 흐름을 만든 점도 운영 효율에 크게 도움이
됐습니다.
