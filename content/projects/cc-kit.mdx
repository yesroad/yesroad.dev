## 배경

Claude Code를 일상 개발에서 활용하면서 매번 같은 컨벤션, 같은 규칙, 같은 워크플로우를 매 프로젝트마다 반복 설정하는 비효율을 느꼈습니다. AI 코딩 도구가 프로젝트마다 일관된 결과를 내려면 rules · agents · skills · commands를 프로젝트마다 활용할 수 있는 구조가 필요하다고 생각했습니다.

## 설계 원칙

Claude Code를 제대로 활용하려면 도구를 잘 쓰는 것보다 도구가 잘 동작하는 환경을 설계하는 게 먼저라고 생각했습니다. 에이전트가 일관되게 움직일 수 있도록 나에게 맞게 조율된 개인 하네스를 만드는 것이 목표였습니다.

AI가 바로 코드를 작성하지 않고, 먼저 현재 코드를 파악한 뒤 필요한 변경을 정리하고, 계획을 세운 다음 구현과 확인까지 이어가도록 작업 순서를 정했습니다. 간단한 작업은 빠르게, 복잡한 작업은 더 세밀하게 진행되도록 조절했습니다.

## 접근 방식

- `/setup` 커맨드로 기술 스택 인터뷰(7문항) 진행 후 프로젝트 맞춤 `.claude/` + `CLAUDE.md` 자동 생성
- 15개 스킬, 5개 에이전트, 7개 커맨드로 구성된 플러그인 시스템 구축
- 작업 범위와 계획을 먼저 정리한 뒤, 구현이 끝나면 코드 품질 검증과 테스트를 거쳐 커밋하는 프로세스 구현
- 훅으로 작업 완료 시 알림 자동 발송
- Claude Code 마켓플레이스에 플러그인으로 배포

## 인사이트

AI 도구를 잘 쓰는 것과 AI 도구를 위한 환경을 잘 설계하는 것은 완전히 다른 문제였습니다. 프롬프트 한 줄 잘 쓰는 것보다 규칙 · 에이전트 · 스킬 · 훅이 맞물려 동작하는 시스템을 설계하는 것이 결과적으로 훨씬 큰 생산성 차이를 만들었습니다.
