## 배경

코코네 소개 홈페이지(B2C)를 새로 구축하는 업무를 맡았습니다. 다국어 지원이 필요했고, 마케팅·기획 팀이 코드 변경 없이 직접 콘텐츠를 수정·관리할 수 있는 구조가 함께 필요했습니다.

## 설계 원칙

다국어 텍스트를 코드에 박지 않고 외부에서 관리할 수 있도록 구조를 분리해, 콘텐츠 수정이 개발 배포와 분리되도록 설계했습니다.

## 접근 방식

- Next.js 기반으로 정적 사이트 구축
- Google Sheets API 연동으로 비개발자가 시트에서 직접 다국어 콘텐츠 수정 가능한 구조 설계
- next-i18next 기반 다국어 환경 구성
- 콘텐츠 변경이 코드 변경 없이 반영되도록 운영 흐름 정립

## 결과

콘텐츠 관리를 위해 매번 개발자에게 요청하던 흐름이 사라지고, 마케팅·기획 팀이 Google Sheets에서 직접 텍스트를 수정하면 배포 없이 반영되는 구조가 자리잡았습니다. 다국어 콘텐츠 변경 시 개발 개입이 불필요해졌습니다.
